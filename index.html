Mobile is the future. Raw as it reads; the mere fact that the web is swallowing our day to day things in to its domain, creating the internet of things, is a tangible tech-trend. Let alone phones, even parking meters, cars and homes and many casual devices are becoming smart by joining this revolutionary proliferation. The central and primary technology that will drive the internet of things for tomorrow and today is the API technology. API Discovery is a platform for App developers to search; discover and learn APIs and start consuming data and services from the APIs. The platform also provides the capability for API providers to register their APIs and tell the world what they offer.

Quoting Gartner on research about Extending the Enterprise with Public Web APIs- "The continued proliferation of devices, form factors and platforms is making it increasingly difficult for development organizations to meet business and user demands for mobile applications."
And then it continues... "By 2014, 75% of the Fortune 1000 will offer public Web APIs." Gartner Nov, 2012.
We have seen some success stories from this tech-evolution, in fact one is here: "Salesforce.com built a large, rich partner ecosystem by opening core services for partners to innovate and extend. Today, more traffic comes through the Salesforce API than through its website. As of mid-2011 more than 60 percent of the traffic to Salesforce.com comes through APIs." -from APIs, by Daniel Jacobson, Orielly, 2012.
The API Discovery Platform is a capability for managing and sharing API data and information. API providers will publish their products (in this case APIs) and API consumers will rate, discuss and share information about their experiences of using the APIs. And as the technology shifts towards APIs, the API Discovery Platform will be the central system that tells you what you can do, how, at what cost and with what level of effort/ease.

FAQ

What is API Discovery?
API Discovery is a platform for managing APIs. From API provider perspective, it is a platform on which the provider can publish APIs. From API consumer perspective, developers, product managers, architects can search, browse, discover, rate and discuss APIs. This makes the search and evaluation of APIs very convenient.
I'm a Software Architect and I design enterprise systems; how can I use API Discovery?
API Discovery is a one stop shop to search for APIs; to rate them; to share experience of how mature a particular API is; cost of integrating an API with your system...etc. The cost of architecting a system is huge; and cost of failure could be massive. Use API Discovery to evaluate the maturity, stability, scalability, ease of integration and many other software architecture metrics.
I am a student. I am an App Developer, I write cool Apps for iPhone and Android, how is API Discovery going to help me?
It is a common and successful design to write Apps that are driven by APIs. In API Discovery, you will search, discover and learn APIs that can give you features that you are looking for. For eg, if you are looking for payment API that you would like to integrate to your App, the platform will give you a set of APIs and their costs, their maturity and ease of integration..etc. Keep in mind that a lot of APIs are free to the public so API Discovery will provide you with the right choice for your App.
